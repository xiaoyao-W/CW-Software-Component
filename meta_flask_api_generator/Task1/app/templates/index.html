Split orders, sync sub-orders, auto-document the logic

After payment, the system splits a paid order into per‑stall sub-orders and synchronizes them to each stall system. On deploy/CI, a versioned flowchart of the splitting logic is generated automatically.

  • 1) Receive Paid Order Event

    A single paid order triggers the splitting pipeline.

  • 2) Split into Sub‑Orders

    Items are grouped by stall; each group becomes a sub-order with references to the parent order.

  • 3) Sync + Versioned Flowchart

    Sub-orders sync to stall systems, and CI emits a versioned flowchart artifact on deploy.
